# LINGAFON — Карта страниц и моделей по ролям

> Черновик для планирования. Бренд: **Lingaphone.kz** — школа английского языка, Алматы (3 филиала) + онлайн по миру. Подробности по продукту см. [lingaphone_research.md](lingaphone_research.md).

---

## Контекст продукта (короткое summary)

- **ЦА:** дети 6-16 лет (решение принимает родитель)
- **Формат:** мини-группы — offline (Алматы, 3 филиала) + online (весь мир)
- **Уровни:** A1-S1 → A2-S2 → S3-S4-B2 → F1-F2 → F3-F4 (5 ступеней)
- **Методика:** British Accent, аудио через специальные наушники, без зубрёжки
- **Главная воронка:** пробный урок → запись в группу → месячная оплата
- **Главная мотивация:** «Маркет Достижений» — каждый месяц ученик получает медаль (🥇/🥈/🥉) за средний балл → реальные деньги (5к / 3к / 1к тг) + накопительные бонусы за подряд идущие золотые месяцы
- **Мероприятия:** лагерь, Halloween, Новый год, Spring Vibes, 8 марта — отдельные платные ивенты

## Базовые модели (предварительная схема)

```
User (общий, как в femo)
  ├─ Student (ребёнок 6-16)
  ├─ Parent
  ├─ Teacher (педагог)
  └─ Admin

Branch              — филиал (Almaty-1, Almaty-2, Almaty-3, ONLINE)
Level               — A1, A2, S1, S2, B2, F1, F2, F3, F4 (enum или таблица)
Group               — уровень + педагог + филиал + расписание + макс. размер
GroupMember         — M:N: Student ↔ Group, дата вступления, статус (ACTIVE/LEFT)
ParentToStudent     — M:N: Parent ↔ Student (можно переиспользовать из femo)

Lesson              — один конкретный урок группы (дата, тема, материалы)
Attendance          — Student ↔ Lesson (PRESENT / ABSENT / LATE)
Grade               — оценка за урок (1-5) с комментарием педагога
Homework            — задание по уроку (опционально)

MonthlyMedal        — агрегат: Student + месяц → средний балл + медаль + payout (вычисляется на конец месяца)
Payout              — факт выплаты денег ребёнку (status: PENDING/PAID, метод)
GoldStreak          — счётчик подряд идущих золотых месяцев (для бонусов 3/6/9)

Application         — заявка с лендоса (пробный урок / запись / вопрос)
TrialLesson         — пробное занятие: ученик + педагог + filial/online + результат теста уровня

Event               — лагерь, праздник, ивент (платный/бесплатный)
EventParticipant    — кто записан на ивент

Payment             — оплата родителем за месяц/пакет/ивент
Package             — пакеты (по уровню × формату × длительности)

Notification, Conversation, Message — можно переиспользовать из femo
```

## Роль 1: Student — `/student/`

```
/student/
├── index.vue              — Dashboard: текущая медаль месяца (прогноз), баланс, ближайший урок, streak золотых месяцев
├── groups/
│   ├── index.vue          — Мои группы (уровень, педагог, расписание, состав)
│   └── [id].vue           — Детали группы: журнал моих оценок, материалы, домашка
├── schedule.vue           — Календарь всех моих уроков (offline filial / online), кнопка «подключиться»
├── grades.vue             — Журнал успеваемости: оценки за каждый урок, средний балл текущего месяца
├── achievements/
│   ├── index.vue          — Маркет Достижений: история медалей, общий заработок, streak
│   └── payouts.vue        — История выплат (когда, сколько, статус)
├── homework/
│   ├── index.vue          — Домашка по урокам
│   └── [id].vue           — Конкретное задание
├── materials.vue          — Аудио для наушников, видео уроков, словари, конспекты
├── events.vue             — Лагерь, Halloween и т.д. (запись, статус)
├── certificates.vue       — Мои сертификаты (после уровней, конкурсов)
└── messenger/             — Чат с педагогом
```

**Tone of UI:** игровая, яркая, как в Duolingo для детей. Прогресс к следующей медали — главный визуал.

## Чего из femo точно НЕ берём

- Student Model / Instructional Model (IAE) — у lingafon методика человеческая (педагог в группе), не адаптивная
- 6 форматов ДЗ с AI-проверкой как основной механизм оценки — у lingafon педагог ставит оценку, AI только для самостоятельной практики
- Сложные траектории и curriculum-планы IAE — у lingafon программа фиксированная по уровням (A1-F4)

---

## Что из femo дополнительно ПЕРЕИСПОЛЬЗУЕМ под lingafon

- **WebRTC lesson room** (`widgets/lesson-room`, `pages/lesson/[id].vue`) — для online-уроков
- **AI-tutor блок** (`widgets/ai-chat`, `features/ai-session`, `server/api/ai/*`) — адаптируем под английский: тренировка произношения через Whisper, диалоги, разбор ошибок
- **Адаптивный движок диагностики** (`features/diagnostics`) — переиспользуем для бесплатного теста уровня на лендосе (но без привязки к Student Model — просто маппинг ответов → A1/A2/.../F4)

## Решения (2026-05-18)

> **Главный режим работы: UI-first.** Сначала рисуем все экраны на мок-данных, потом подключаем БД и логику. Не залипаем в схемы и интеграции пока интерфейс не утверждён.

1. **Выплаты медалей** → пока только UI (история медалей, баланс, история выплат). Реальный перевод денег и интеграция с банком — позже, после того как интерфейс утверждён.

2. **Оффлайн-уроки** → ведём в платформе. Журнал и посещаемость одинаково и для offline, и для online групп. Педагог каждый день заходит и отмечает.

3. **Видео онлайн-уроков** → как в femo. Своя WebRTC-комната (`/lesson/[id]`) + доска + запись. Переиспользуем `widgets/lesson-room` из femo.

4. **Админ** → пока один централизованный админ с полным доступом ко всему. Разделение «филиал-админ vs центр-админ» отложили.

5. **AI** → нужен. Делаем AI-компонент для самостоятельной практики между уроками: тренажёр произношения (Whisper → анализ), диалоги, проверка письменных заданий, AI-разбор ошибок. По функционалу — порт `widgets/ai-chat` + `features/ai-session` + `server/api/ai/*` из femo, адаптированных под изучение английского.

6. **Тест уровня на лендосе** → **бесплатный публичный тест** (`/level-test`). Доступен без регистрации. Результат **сохраняется анонимно** (по `localStorage` или анонимной сессии Supabase), чтобы админ видел воронку: кто сдавал, какой уровень, когда (и при последующей регистрации привязывается к юзеру).

   **Модель:** `LevelTestAttempt { id, anonId, userId?, answers, computedLevel, completedAt, source }`.
   Админ-экран: `/admin/analytics/level-tests.vue` — лиды без регистрации.

7. **Админ-роли** → один тип admin. Без подролей.
