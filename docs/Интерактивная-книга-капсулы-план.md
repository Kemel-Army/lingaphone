# Интерактивная книга (капсулы «Мой путь») — план портирования под английский

> Источник-референс: `b1411/arna-tutor` @ `femo` (SHA `46fa8d6`). Стек-близнец: тот же Nuxt 4 + Supabase + FSD + Femi-маскот. Цель: перенести систему «Мой путь / капсулы» и адаптировать под **английский язык**, где контент капсул **AI-генерируется из загруженной книги** (черновик) + **правится вручную** в админке.

## 0. Зафиксированные решения

- **Глубина:** полный порт капсул arna — 11 слоёв, `CapsuleRunner`, AdventureMap, серверный скоринг.
- **Контент:** AI-черновик из PDF книги → ручная правка (admin content CRUD).
- **Предмет:** английский (не математика). Каркас слоёв предметно-нейтрален; math-специфика замещается языковой.
- **Врезка:** книга «материализуется» набором тем (`PathTopic.bookId`).

## 1. Ключевой вывод разведки

Lingaphone = **сестринский форк** arna. Локально **уже есть** весь анимационный субстрат:
`FemiMascot`, `ThemeBackground`, `useConfetti` (burst/trophy/bossWin/streakSparkle), `useMascotReactions` (flash/flashState/flashLine), `useTTS`, `useIdleDetector`, `StreakCounter`, `useAnimatedCounter`, auto-animate, canvas-confetti.
Локально **НЕТ** самой системы learning-path/capsule (чисто, конфликтов нет) — переносим целиком.

## 2. Модель данных (5 таблиц Supabase)

Иерархия: **Book → PathTopic → PathLesson (капсула) → 11×CapsuleLayer**; прогресс: `PathProgress` (капсула) + `LayerProgress` (слой).
Enum-поля = `TEXT + CHECK` (как в arna, не нативные PG-enum).

### Миграция (эскиз DDL, адаптировать RLS под локальную конвенцию `auth.uid()::text` / `Student.userId`)

```sql
-- PathTopic: тема-раздел книги
create table public."PathTopic" (
  id uuid primary key default gen_random_uuid(),
  "bookId" uuid references public."Book"(id) on delete cascade,   -- НОВОЕ: связь с книгой
  "moduleId" uuid references public."Module"(id) on delete set null, -- опц.: тема ↔ модуль книги
  name text not null,
  description text,
  level text,                        -- A1|A2|B1|B2 (вместо gradeLevel)
  icon text default 'i-lucide-book-open',
  color text default 'green',
  "orderIndex" int default 0,
  "totalXp" int default 0,
  "durationMinutes" int default 0,
  "createdAt" timestamptz default now(),
  "updatedAt" timestamptz default now()
);

-- PathLesson: капсула (урок = 11 слоёв)
create table public."PathLesson" (
  id uuid primary key default gen_random_uuid(),
  "pathTopicId" uuid not null references public."PathTopic"(id) on delete cascade,
  title text not null,
  subtitle text,
  "orderIndex" int default 0,
  "durationMinutes" int default 15,
  "xpReward" int default 50,
  "masteryThreshold" int not null default 80 check ("masteryThreshold" between 50 and 100),
  difficulty text not null default 'STANDARD' check (difficulty in ('LIGHT','STANDARD','DEEP')),
  "createdAt" timestamptz default now(),
  "updatedAt" timestamptz default now()
);

-- CapsuleLayer: 11 слоёв на капсулу
create table public."CapsuleLayer" (
  id uuid primary key default gen_random_uuid(),
  "lessonId" uuid not null references public."PathLesson"(id) on delete cascade,
  "layerType" text not null check ("layerType" in
    ('HOOK','DIAGNOSTIC','INTUITION','EXPLANATION','FORMALIZATION',
     'WALKTHROUGH','TRAINER','SCENARIO','TRAPS','TEACH_BACK','MASTERY_CHECK')),
  "orderIndex" int not null check ("orderIndex" between 1 and 11),
  title text not null,
  subtitle text,
  icon text not null default 'i-lucide-sparkles',
  "accentColor" text not null default 'green',
  "estimatedMinutes" int not null default 3,
  "xpReward" int not null default 10,
  content jsonb not null default '{}',              -- shape дискриминируется layerType
  "completionCriteria" jsonb not null default '{}', -- {minAccuracy,minCorrect,minInteractions,minLength,...}
  "createdAt" timestamptz default now(),
  "updatedAt" timestamptz default now(),
  unique ("lessonId","orderIndex"),
  unique ("lessonId","layerType")
);

-- LayerProgress: прогресс по слою (на ученика)
create table public."LayerProgress" (
  id uuid primary key default gen_random_uuid(),
  "studentId" uuid not null references public."Student"(id) on delete cascade,
  "layerId" uuid not null references public."CapsuleLayer"(id) on delete cascade,
  "lessonId" uuid not null references public."PathLesson"(id) on delete cascade,
  status text not null default 'LOCKED'
    check (status in ('LOCKED','AVAILABLE','IN_PROGRESS','COMPLETED','FAILED')),
  score int check (score is null or score between 0 and 100),
  "maxScore" int,
  attempts int not null default 0,
  "timeSpentSeconds" int not null default 0,
  "xpEarned" int not null default 0,
  "interactionData" jsonb not null default '{}',
  "startedAt" timestamptz, "completedAt" timestamptz,
  "createdAt" timestamptz default now(), "updatedAt" timestamptz default now(),
  unique ("studentId","layerId")
);

-- PathProgress: агрегат по капсуле
create table public."PathProgress" (
  id uuid primary key default gen_random_uuid(),
  "studentId" uuid not null references public."Student"(id) on delete cascade,
  "pathLessonId" uuid not null references public."PathLesson"(id) on delete cascade,
  "currentLayerIndex" int not null default 1 check ("currentLayerIndex" between 1 and 11),
  "layersCompleted" int not null default 0 check ("layersCompleted" between 0 and 11),
  "masteryScore" int check ("masteryScore" is null or "masteryScore" between 0 and 100),
  "masteryMaxScore" int,
  "masteryAchieved" boolean not null default false,
  "xpEarned" int not null default 0,
  "lastActivityAt" timestamptz, "completedAt" timestamptz,
  "createdAt" timestamptz default now(), "updatedAt" timestamptz default now(),
  unique ("studentId","pathLessonId")
);
```

**RLS** (как в arna, адаптировать под локальный `get_current_user_id()` / `auth.uid()::text`):
- `PathTopic`/`PathLesson`/`CapsuleLayer`: `SELECT` для `authenticated USING (true)`; `ALL` для `service_role`.
- `PathProgress`/`LayerProgress`: CRUD только по своим строкам (`studentId → Student.userId = current user`); read для родителя/учителя; `ALL` для `service_role`.

**Отличия от arna:** `gradeLevel`→`level (A1..B2)`; `subjectId`→убрать или сделать nullable (единый предмет = английский); **добавлен** `bookId` (+ опц. `moduleId`) — книга разворачивается в темы.

## 3a. Конкретный маппинг книги — Access Grammar 1 (Express Publishing, Evans/Dooley)

Загружено: `Access_1_GB.pdf` — 78 стр, **скан-картинки** (JPEG2000/JBIG2, нет text-слоя → для AI нужен OCR или vision-LLM по отрендеренным страницам). Рендер: pdfjs + `@napi-rs/canvas` + wasm-декодеры (`pdfjs-dist/wasm/`, флаг `wasmUrl`) — работает.

**Структура (из оглавления):** 10 модулей → **28 юнитов** + per-module «Exploring Grammar» + «Revision» + приложение «Irregular Verbs».

| Book | → | Наша модель |
|---|---|---|
| Module (10 шт: «Module 1»…) | → | `PathTopic` (10 островов на карте) |
| Unit (28 шт: «1 The Indefinite Article a–an», «8 Some/Any»…) | → | `PathLesson` (капсула, 11 слоёв) |
| «Exploring Grammar (Units X-Y)» / «Revision (Units 1-N)» | → | доп. `PathLesson` типа review/mastery |
| Irregular Verbs | → | отдельная тема/справочник |

**Анатомия юнита (консистентна везде):** бейдж-номер юнита → **rule-box** (буллеты или таблица, напр. Countable/Uncountable × Affirmative/Negative/Interrogative) → нумерованные **упражнения с фото** (fill-in `a/an`; categorize в колонки A/An; write sentences `is/isn't/are/aren't` по картинке) → иногда **Speaking**.

**Упражнение → слой:**
- rule-box (таблица/буллеты) → **FORMALIZATION** + **EXPLANATION** (правило + примеры + фото-chunks).
- fill-in / choose / rewrite → **TRAINER** (choice/input/tap-pair).
- picture+word (фото предметов) → **INTUITION** `word-picture-match` (фото уже в книге, вырезаем).
- Speaking task → **SCENARIO** (мини-диалог) / **TEACH_BACK** (проговори правило).
- «Revision» секция модуля → **MASTERY_CHECK**.
- HOOK/DIAGNOSTIC → AI генерит крючок + пред-проверку по теме юнита.

**Фото из книги:** каждая страница = скан; для `word-picture-match` / `ExplanationChunk{image}` / HOOK — вырезаем регионы страниц (bbox изображений через pdfjs OperatorList, либо кроп отрендеренного PNG) → Storage → URL в `content`. Это и есть «фотографии с книги приложены».

**Вывод:** грамматический УМК с жёсткой структурой = **лучший случай для AI-генерации**: rule-box даёт FORMALIZATION почти дословно, упражнения → TRAINER/INTUITION 1:1, Revision → MASTERY. Vision-LLM (скан → structured JSON по `CapsuleLayerContent`) — основной путь (text-слоя нет).

## 4. 11 слоёв под английский

Каркас 1:1, `content`-shape тот же (`app/entities/learning-path/model/types.ts`), меняется наполнение:

| Слой | В arna (math) | Под английский |
|---|---|---|
| HOOK | эмодзи-комикс, крючок | то же: кадр из темы книги + вопрос-крючок |
| DIAGNOSTIC | mcq/tap-correct | что уже знаешь: mcq по vocab/grammar темы |
| INTUITION | math-манипуляторы (fraction/clock) | **языковые виджеты** (см. ниже): word↔picture, sentence-order, listen-and-pick |
| EXPLANATION | chunks + KaTeX | chunks: text/callout/**image (фото из книги)**/comic/tap-reveal; grammar-объяснение |
| FORMALIZATION | формулы, anatomy | **grammar-rule**: структура (напр. `Subject + do/does + V`), термины (Present Simple…) |
| WALKTHROUGH | разбор в столбик | разбор эталонного примера: собрать предложение по шагам |
| TRAINER | choice/numeric/tap-pair | choice/**input**/tap-pair — идеально для vocab (слово↔перевод), grammar-дрилл |
| SCENARIO | кафе/космос, math-заказы | **ролевой диалог** (в кафе закажи еду, в аэропорту…): темы-сцены переиспользуем |
| TRAPS | типичные math-ошибки | типичные англ-ошибки (`he don't` → `he doesn't`), flip/spot/fix-it |
| TEACH_BACK | объясни математику | **speaking**: расскажи/перескажи (Whisper/Web Speech + `/api/ai/evaluate-teachback`) |
| MASTERY_CHECK | итог-тест | итог-квиз по теме, трофеи gold/silver/bronze, share-карточка |

**Языковые INTUITION-виджеты (новые, заменяют math):**
- `word-picture-match` — сопоставь слово ↔ картинку (из книги).
- `sentence-builder` — собери предложение из слов-плиток (порядок слов).
- `listen-and-pick` — послушай (TTS) → выбери правильное написание/картинку.
- `gap-fill` — вставь пропущенное слово.
(Math-виджеты `fraction-trio/clock/protractor/…` не переносим или оставляем заглушкой — для англ. не нужны.)

`renderMath` (KaTeX) для англ. почти не нужен → оставляем как опциональную утилиту (пусть будет, но контент его не требует).

## 8. AI-генерация из книги (НОВОЕ — greenfield)

В arna нет — проектируем. Конвейер:

> ⚠️ Access Grammar 1 — **скан без text-слоя** → text-экстракция не работает. Основной путь = **vision-LLM по отрендеренным страницам** (pdfjs+canvas → PNG → GPT-4o vision → structured JSON). Для книг с text-слоем — дешёвый text-путь.

```
PDF книги (books bucket)
  → рендер страниц в PNG (pdfjs + @napi-rs/canvas + wasmUrl) [+ text-слой если есть]
  → vision-LLM пайплайн (OpenAI, server route /api/ai/generate-capsules):
      1) сегментация: PDF → units/topics → PathTopic[] (name, level, order)
      2) на тему → 1..N PathLesson (капсулы)
      3) на капсулу → 11 CapsuleLayer.content JSON по TS-схеме layerType
         (function-calling / JSON-schema, привязка фото-страниц как image-chunks)
  → сохранить как draft (isPublished=false) → админ правит → publish
```

- **Схема ответа LLM** = ровно `CapsuleLayerContent` из `types.ts` (11 discriminated shapes) → валидация Zod на сервере, ретрай при несоответствии.
- **Фото из книги:** страницы PDF → PNG в Storage → URL в `ExplanationChunk{kind:'image'}` / HOOK-кадры / word-picture-match.
- **Стоимость/асинхронность:** генерация тяжёлая → фоновая задача (Nitro task / очередь), статус на карточке книги в админке.
- **Итеративно:** сначала генерим EXPLANATION+TRAINER+MASTERY (ядро), остальные слои — второй проход.
